# Contributing to MA EV ChargeMap

Thank you for your interest in MA EV ChargeMap! This is a **personal portfolio project**, but feedback and suggestions are welcome.

---

## Reporting Issues

If you find a bug or have a suggestion:

1. **Search existing issues** to avoid duplicates
2. **Open a new issue** with:
   - Clear description
   - Steps to reproduce (for bugs)
   - Expected vs. actual behavior
   - Screenshots if applicable

---

## Suggesting Enhancements

Enhancement ideas are welcome! Please open an issue with:
- **Use case**: What problem does this solve?
- **Proposed solution**: How would it work?
- **Alternatives**: Other approaches you considered

---

## Code Contributions

While this is primarily a solo project, I may accept pull requests for:
- Bug fixes
- Documentation improvements
- Test coverage improvements
- Code quality enhancements

### Before Contributing

1. Open an issue to discuss the change
2. Wait for approval before implementing
3. Follow existing code style and patterns

### Development Setup

See README.md for setup instructions.

### Pull Request Process

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/your-feature`)
3. Make your changes
4. Add/update tests as needed
5. Update documentation
6. Commit with clear messages
7. Push to your fork
8. Open a pull request

### Code Standards

**Python (Backend)**:
- PEP 8 style
- Type hints for functions
- Docstrings for public APIs
- pytest for tests

**TypeScript (Frontend)**:
- ESLint recommended rules
- Type annotations required
- React best practices
- Descriptive component names

### Testing

- Add tests for new features
- Ensure existing tests pass
- Run: `pytest backend/tests/`
